웹 프로그램 통신
모든 웹프로그램은 사용자가 웹 브라우저를 이용해 웹프로그램이 가지고 있
자원(상품 정보, 강좌목록)을 요청하면 웹브라우저가 이해할 수 있는
형태로 재가공 또는 그대로 웹브라우저에 반환


웹서버와 웹브라우저간 발생하는 자원 반환 단계에서는 웹서버가
 콘텐츠 협상(contents negitiation)
단계 거친 후 웹 브라우저에 결과 반환
 웹브라우저가 무엇을 처리할 수 있는 웹서버와 협상하는 단계

 3가지 단계

1. 서버 기반의 협상
2. 에이전트 기반 협상
3. 투명한 협상(1,2 혼합한 협상)

 서버기반 협상: 웹서버가 웹브라우저에 반환할 데이터의 형태를 직접 결정

 에이전트(캐시서버) 기반의 협상: 웹 서버가 응답할 데이터 처리 형태를 결정하기위해 첫 번째 수신을 
 처리한 에이전트에 의해 형태를 결정

협상에 필요한 HTTP 메시지 헤더  // Key : Value .. Host : 서버주소 ~~
Accept : 브라우저가 처리할 수 있는 데이터의 형태, 선호도 (text/html)

Accept Language : 브라우저가 수용할 수 있는 응답결과의 언어와 선호도
Accept Encoding : 브라우저가 수용할 수 있는응답 인코딩

HTTP 메세지는 요청(Request)메세지와 응답 (Response)메세지로 나눔

Http 요청 메시지 : 첫줄 메소드(GET, POST) 이름, 요청 자원 주소, HTTP 버전(HTTP/1.1), 호스트명, 웹브라우저가 무엇인지(<키>헤더: User_Agent[Mozilla]) 
어떤 언어(ko_KR,ko;en-US)/ 자원 형태를 받아들이는지 기록

HTTP 응답 메세지 : 첫행 HTTP 버전  HTTP 상태 코드(200:성공)==>사용자가 요청한 내용 서버에서 처리를 잘 된것을 반환, 상태코드 문자열 OK
두번째 행부터는 순서 없이 정보를 기술 (날짜, 서버 종류, 사용자 정의 헤더X-시작하는 헤더명---> 사용자 정의 헤더) 
사용자 정의 헤더는 웹프로그램과 웹 브라우저가 해석할 수 있을 때 의미있는 정보가 된다.
